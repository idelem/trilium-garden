moderate web clipping

今天看了这篇文章 Evernote - 遗毒与缺陷，其中有一段提到 evernote 的遗毒之一，web clipper：

对 Evernote 的用户来说，做出这种扩展等于引诱他们疯狂保存网页，因为多数人其实没有多少东西要记录，再加上 Evernote 那个编辑器很不适合自己写东西，web clipper 就成了积累内容的最速最佳途径。而且正常人都是没有足够克制力的，这就像把冲锋枪交给一群小孩子玩，最后一定会出事故。相当多的 Evernote 用户都在讲述这样的经验——自己的 Evernote 被 web clipper 搞成了一个垃圾场，甚至最后已经到了无法整理的地步，只能放弃用了多年的 Evernote 转投其它软件。Evernote 的合作生态也是越做越大，很多网站和 app 的内容都可以一键发送到 Evernote，所以世界上的垃圾场就越来越多了。

（不用说，我当然是立刻 clip 了这一段）

我以前用过 evernote，而且也像这段所说的一样，把它搞成了垃圾场，最后不得不星夜逃离，从此过上了东挑西拣的笔记流亡生活。

现在，我还是在用 trilium 的 web clipper，但是一切已经不同了：trilium web clipper 是我目前用过的最好的剪辑工具。trilium 也没有变成垃圾场。

为什么？

这是因为，大部分剪辑产品设计都忽略了一点：用户根本不需要剪辑整页笔记！

trilium web clipper 的设计很简单：

  • 选中一个段落，把它保存到当天的日记下，同时存储原网页链接
  • 如果当天保存过这个页面上的段落，就把新剪辑附加到那则笔记上（不会制造太多条目，一天一页一条）
  • 可以保存链接
  • 可以保存图片

这么做的好处是：

  • 分几天阅读长文章，每天的摘抄都在同一页，摘抄下面还可以写自己的笔记
  • 全部读完后，用「搜索结果」或者「book」类型的笔记把它们全汇总在一棵树下
  • 可以把段落 clone 到它应该去的地方，归档不费吹灰之力

i call this practice moderate web clipping. it helps build:

  • a zettelkasten
  • a private web ready for xanalogical structures and transclusion
